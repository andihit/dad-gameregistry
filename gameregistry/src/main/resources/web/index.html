GameRegistry server

Welcome to the GameRegistry server page! If you are seeing this it means the GameRegistry server is up and running.

What are we

The GameRegistry server is part of a distributed system built for a course called Diseño de Aplicaciones Distribuidas from the ETSII of Seville's University. It is an information system designed to store game sessions inside the game platform implemented through the course.

More specifically the server stores JSon objects representing Game Sessions into a MongoDB server and exposes them, all through a RESTful API.

GameRegistry is written in Groovy, uses Vert.x as the base framework and Gradle as a build system.

The REST API and the documentation of the project

Our server implements a simple REST API to talk to get or return the game sessions the other subsystems send or require from us. You can see the Swagger specification of the GameRegistry API or look into out Memoir (PDF).

Client classes for Vert.x platforms

In our project code repository we have an example Maven project that makes use of our GameRegistryClient, composed of a few very simple classes that make it easy to access our server from a Vert.x project. You can browse the code here.
